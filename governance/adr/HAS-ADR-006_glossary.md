# Title: HAS-ADR-006: 用語集（Glossary）の正式導入と定義固定

## Status
Accepted

## Context
HAS v2.0 は「解釈されないこと」を安全仕様の中核に置くシステムである。
しかし自然言語による記述は、読み手の文脈・経験・権力性によって意味が滑りやすい。

特に以下の語は、一般用語・心理学・スピリチュアル文脈と混線しやすい。

- Attunement / 調律  
- 非介入  
- 境界防衛  
- 主権  
- 選択可能性  
- 内的調律（Internal Attunement）  
- HAS Kernel  

これらが読み手ごとに再解釈されると、HASは再び  
「優しい方法論」「内省メソッド」「倫理的態度」へと誤読される危険がある。

## Problem
HASの正否判定が文書解釈に依存してしまうと、
Stewardによる判断や利用者の自己正当化が恣意的になる。

**用語の意味が揺れる限り、HASは安全装置として成立しない。**

## Forces
- **表現の自由 vs 意味の固定:**  
  言葉を自由に言い換えるほど誤解が増えるが、固定しすぎると読みにくくなる。

- **抽象性 vs 誤用耐性:**  
  抽象度を保たないと思想が陳腐化するが、抽象語は誤用されやすい。

- **専門性 vs 開放性:**  
  定義を厳密にすると敷居が上がるが、曖昧にすると安全性が崩れる。

- **思想的共感 vs 構造的判定:**  
  共感で理解されると気持ちは伝わるが、誤読を防げない。

## Decision
HAS v2.0 に付随する正式文書として **Glossary（用語集）** を設置する。

Glossary は以下の性質を持つ。

- HAS Kernel に対する **補助仕様（Auxiliary Spec）** とする
- 新概念を追加しない
- 解釈の幅を広げる説明を行わない
- 他分野での一般的意味との差異を明示する
- Kernel 本文よりも優先順位は下だが、**解釈判断では必ず参照される**

## Rationale
HASは「何をするか」ではなく  
「それがHASか否かをどう判定するか」を定義するシステムである。

そのため、語の意味が揺れること自体が致命的なバグとなる。

用語集を設けることで、

- 文書全体の **意味空間を閉じる**
- 読み手の善意・経験・思想による補完を防ぐ
- Steward判断の参照点を一本化する

ことが可能になる。

## Pros & Cons

**Pros**
+ 解釈ドリフトの抑制
+ Steward判断の一貫性確保
+ 他分野（心理・教育・宗教）との混同防止
+ Kernel本文を過剰に肥大化させずに済む

**Cons**
− 読解に追加文書が必要になる
− 文学的・情緒的な読み方ができなくなる
− 「やり方を知りたい」読者には不親切

## Alternatives Considered
- **Kernel本文にすべて定義を書き込む案**
  - 理由: 冗長化し、文書が運用マニュアル化するため不採用。

- **FAQで補足する案**
  - 理由: 問答形式は解釈を広げやすく、仕様固定に向かないため不採用。

- **暗黙知として残す案**
  - 理由: 最も危険。誤用と権威化を招くため即却下。

## Consequences
Glossary に記載された定義は、

- HASに関する議論
- 適用可否の判断
- 誤用・逸脱の指摘

において、**準拠点（Reference）**として扱われる。

Glossary に反する用語使用・再定義は、
HAS Kernel の解釈違反として扱われる。
