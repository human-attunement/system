# ADR-021: Attunement Map における「位置識別」と「HAS外条件」の明示

## Status
Accepted

## Context

HAS v2 系の整理において、従来の「段階（Levels）」モデルは、

- 成長モデルとして誤読されやすい
- 正しく使おうとする努力が「操作」や「完遂圧」を生みやすい

という問題を抱えていた。

また、実践上、

- すでに判断している
- 完遂を目指して動いている
- 介入を固定している

にもかかわらず、「HASのどこかの状態にいる」と誤認されるケースが想定された。

これらはHASの設計思想（非操作・非完遂・非最適化）と根本的に矛盾する。

## Problem

- HASが「状態の地図」であるにもかかわらず、  
  使用者が **HASの外にいる状態**を内部状態として誤認してしまう。
- 善意や熟達によって、HASが「正しく使う技法」へ変質するリスクがある。
- 失敗や乱れと、HAS外状態の区別が曖昧になる。

## Forces

- 説明を簡潔にしたい欲求
- 実践者からの「具体的にどうすればいいか」という圧力
- 既存の成長段階モデルへの無意識な引き寄せ
- 「役に立たなければ意味がない」という外在的評価軸

## Decision

1. Attunement Levels を廃止し、**Attunement Map（位置の識別）**へ移行する。
2. HASの4状態（観察／判断保留／仮動／乱れ）は  
   **HAS内でのみ成立する位置**であることを明示する。
3. 以下を満たさない場合は **HAS外条件** と定義する：
   - 結論を確定していない
   - 完遂を目的化していない
   - 介入を「やるべきこと」として固定していない
4. HAS外にいる場合は、状態判定より先に  
   **「観察に戻る判断」そのものを最初の実践**と位置づける。

## Rationale

- HASは「うまくやる体系」ではなく、「壊れないための体系」である。
- HAS外条件を明示しない限り、誤用は必ず善意から発生する。
- 「HAS外にいる」こと自体を失敗としない設計が、実践継続性を担保する。
- 図にHAS外を含めないことで、HAS外を「別の状態」と誤認させない。

## Pros & Cons

### Pros

- \+ 善意による誤用を構造的に防止できる
- \+ 実践者が「戻る」判断を正当化できる
- \+ 熟達や正しさ競争が起きにくい
- \+ HASの非操作性が明確になる

### Cons

- − 「何もできない体系」に見える可能性
- − 即効性や成果を求める利用者には不親切に映る
- − 技法や対処法を期待する層には物足りない

## Note

- HAS外条件は「排除」ではなく**安全ゲート**として扱う。
- HAS外状態への具体的対処（例：展開済みArmorの扱い）は、
  Failure Modes または Techniques で後続定義する。
- Attunement Map は「使い方」ではなく「誤解防止装置」である。
- **HAS外にいることは失敗ではない。気づいた時点で観察に戻ればよい。**

## Alternatives Considered

1. HAS外条件を暗黙知として扱う  
   → 誤用リスクが高すぎるため不採用。
2. HAS外も含めた拡張状態モデルを作る  
   → HAS外を正当な位置と誤認させるため不採用。
3. ファシリテータ熟達でカバーする  
   → 個人能力依存となり設計思想に反するため不採用。

## Consequences

- HASは「成果を出す体系」ではなく「連鎖を止める体系」として明確化される。
- 離脱・中断・未使用が正当な選択肢として位置づけられる。
- 後続文書（Failure Modes / Techniques）での補完が前提となる。
