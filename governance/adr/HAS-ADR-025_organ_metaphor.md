# Title: ADR-0025: 器官メタファによる自己調節構造の記述導入

## Status
Accepted

## Context
HASではこれまで、「調律」を価値観・態度・手法として誤読されるリスクがあった。
特に以下の誤解が繰り返し発生していた。

- 調律が「新しい能力」「獲得すべきスキル」として読まれる
- 調律が「より良い上位状態」「成熟段階」として理解される
- 適応・適合・即興との違いが、行為や性格の差として捉えられる

これらは、HASのKernelにある「人間は最初から完全である」「何かを作らない」という存在論と矛盾する。

そのため、**能力論・成長論・価値論に回収されない記述軸**が必要になった。

## Problem
調律を、
- 技法
- 態度
- 成長段階
としてではなく、

**すでに内在している働きとの関係構造**として、
誤読なく記述するには、どのような概念枠が必要か。

## Forces
- 調律を説明しようとすると、Doing / スキルの話に落ちやすい
- 比喩を使わないと直感的理解が困難
- しかし比喩は、機能・正しさ・方向性を定義してしまう危険もある
- Concept / docs / quickref 全体で一貫した説明軸が必要
- Quick Reference では短く、Concept では構造的に説明できる必要がある

## Decision
調律を含むHASの中核概念を、  
**「人が元から持つ内在的な自己調節器官」との関係構造**として記述するため、  
器官（Organ）メタファを導入する。

## Rationale
- 器官メタファは「獲得」ではなく「存在」を前提にできる
- 能力の優劣や成長段階の議論を構造的に回避できる
- 適応・適合・調律を「別の器官」ではなく  
  **同一器官の異なる様相**として説明できる
- 意志の役割を「操作」ではなく  
  **位置・距離・余白**として定義できる
- Concept では構造説明に、Quick Reference では短い定義に展開可能

## Pros & Cons

**Pros**
- \+ 調律がスキル化・技法化される誤読を防げる  
- \+ 「人を変える」「育てる」構造に落ちない  
- \+ 適応・適合・即興との違いを、行為ではなく構造で説明できる  
- \+ Kernel（完全性・非獲得）との整合が取れる  

**Cons**
- − 「器官」という語が生物学的・医学的に読まれる可能性  
- − 抽象度が上がり、初見読者には一度で理解されにくい  
- − 比喩であることを明示しないと、機能論に誤読される恐れ  

## Alternatives Considered
- **能力・スキル比喩**
  - 不採用理由: 成長論・訓練論に回収される
- **OS / システム比喩**
  - 不採用理由: 最適化・設計論に引き寄せられる
- **心理状態モデル**
  - 不採用理由: 正常／異常・段階論を生みやすい

## Consequences
- Concept において、調律は「行為」ではなく  
  **器官と意志の位置関係**として説明される
- docs 全体で、「育てる」「高める」表現を避ける判断基準が明確になる
- Quick Reference では  
  「器官が働ける余地を保つ」という短い定義に集約できる
- 将来的にHASが拡張されても、能力論への逸脱を検知しやすくなる
