# HAS Public Relations Protocol
**HAS広報プロトコル：設置と境界の作法**

## 1. Core Philosophy
**「拡声器」ではなく「展示台」として使う。**

* 広めるためではなく、探している人が見つけられるように「置く」。
* 成功指標は「拡散数（Buzz）」ではなく「誤解の少なさ（Accuracy）」である。
* HASは、ユーザーの購買意欲や欠乏感を刺激してアクセスさせるものではない。

## 2. Posting Rules (Do's & Don'ts)

| Category | Action | Example / Mindset |
| :--- | :--- | :--- |
| **Do (やる)** | **事実と定義を書く** | 「これはXXなシステムです」 |
| | **限界と不向きを書く** | 「XXな人には役に立ちません」 |
| | **離脱の自由を保証する** | 「読まなくていいです」「スルーしてください」 |
| **Don't (やらない)** | **変化を約束する** | $\times$「人生が変わります」「救われます」 |
| | **正しさを主張する** | $\times$「今の時代に必須」「学ぶべき」 |
| | **扇動・共感を狙う** | $\times$「拡散希望」「ピンときた人はRT」 |

## 3. The "Anti-Clickbait" Check
投稿ボタンを押す直前に、以下の問いにイエスと答えられるか確認する。

> **「この投稿を見た人は、『読まない自由』を完全に保持しているか？」**

* **No:** 不安を煽っている、正義感を刺激している、欠乏感を突いている。
    * 👉 **Reject (投稿禁止)**
* **Yes:** 読むか読まないかが、完全に相手の恣意に委ねられている。
    * 👉 **Post (設置)**

## 4. The Exit Clause (離脱条項)
すべてのコンテンツ、すべての接点の末尾に、以下のスタンスを明示する。

> **"If not now, you can come back to it later."**
> 今でなくても、必要なときにまた戻ってきてください。


## 5. 発信の原理原則

**（設置としての発信）**

HASの発信は、広めるための行為ではない。行動を促すための設計でもない。
**ただ、存在を隠さずに置いておくこと**を目的とする。

---

### 発信の目的

HASの発信の目的は、理解されることでも、採用されることでもない。
ましてや、変化や成果を生むことでもない。

**壊れる前に、止まれる座標を失わせないこと。**  
それ以上の目的は持たない。

---

HASにおいて問題となるのは、「発信すること」ではない。**発信が、人の選択を狭め始めること**である。

したがって、以下を満たす限り、発信は許容される。

発信と操作の境界は以下で定義づけられる。

- 非誘導性: 行動を誘導しない
- 非扇動性: 欠乏感や不安を刺激しない
- 非即時性: 判断や選択を急がせない
- 非強制性: 「見なければならない理由」を作らない 

**見た人が、何もしなくていい自由を失わないこと。**  
これが唯一の条件である。

---

### 宣伝と露出の違い

HASは宣伝しない。宣伝とは、行為を起こさせるための設計である。
HASが行うのは露出である。露出とは、**存在を隠さないという選択**にすぎない。

- 「見てほしい」と言わない  
- 「必要だ」と言わない  
- 「役に立つ」と言わない  

ただ、「ここにあります」とだけ示す。

---

### 誤解・権威化に対する態度

HASは、誤解されうる。誤用されうる。権威化されうる。

それらを**防ごうとしない**。

なぜなら、

- 誤解を正したい衝動  
- 正しく理解させたい衝動  
- 変な使われ方を防ぎたい衝動  

これらはすべて、**介入を正当化する Pressure**だからである。

HASが守るのは、

- 定義  
- 境界  
- 離脱の自由  

それだけで十分である。

誤解や権威化は、**制御対象ではない**。

---

### 観測と操作の区別

HASは、完全な不可視を選ばない。必要最低限の観測は許容する。

ただし、観測結果を以下に接続しない。

- 最適化  
- 改善  
- 成果評価  
- 誘導設計  

**測ること自体ではなく、測った結果で何かをいじりたくなること**が境界である。

---

### 発信設計原則

どの媒体にも依存せず、発信時の設計原則を列挙する。

発信設計原則
: 非完遂要請性（途中で離脱してよい）
: 非獲得要請性（何も持ち帰らなくていい）
: 非有用要請性（役に立たなくてよい）

これらをまとめて**非獲得要請性**とし、発信設計にはこれらを組み込む。

発信の形式は問わない。

- 文章  
- 動画  
- ワークショップ  
- 公開イベント  

これは演出ではなく、**設計上の非常口**である。

---

### 成功の定義

HASの発信は、成功を測らない。

- 広まったか  
- 理解されたか  
- 採用されたか  

これらは評価軸ではない。

**誰にも守られなくても、壊れないこと。** それだけが成功条件である。

---

### 結語

HASは、伝える体系ではない。導く体系でもない。

**人が、自分の選択を引き受けられる余地を壊さないための構造**である。

発信とは、その構造を**見える場所に置いておく行為**にすぎない。

それ以上のことは、しない。
